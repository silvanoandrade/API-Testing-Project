QA Analysis – API Testing Project

This repository contains the Quality Assurance analysis performed during Sprint 4 of a QA training project. The objective is to validate the functionality, reliability, and error handling of several API endpoints through structured test design and execution.
The project focuses on ensuring that the API behaves according to the specified requirements by executing both positive and negative test scenarios, documenting the results, and reporting any detected defects.

Project Objectives

Validate API endpoint functionality according to requirements
Identify functional defects and unexpected behaviors
Test both valid and invalid input scenarios
Document test cases and results in a structured format
Report and track issues using a bug tracking system

Scope of Testing

The QA analysis focuses on testing several API endpoints related to product and order management. The validation includes:
Request and response verification

Input validation
Error handling
Response status codes
Edge cases and invalid data scenarios

Test Strategy

The testing process followed these steps:

Requirement analysis to understand expected API behavior
Test case design covering both functional and negative scenarios
Test execution using API requests
Result documentation for each test case
Bug reporting for failed tests or unexpected responses

Test Types Covered

Functional testing
Negative testing
API response validation
Status code verification
Input validation testing

Tools Used

API testing tools (e.g., Postman)
Jira for bug tracking
Excel for test case documentation
GitHub for version control and project documentation
